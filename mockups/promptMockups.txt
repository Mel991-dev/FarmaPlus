CONTEXTO DEL PROYECTO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Sistema: FarmaPlus CRM — sistema de gestión integral para una
droguería pequeña de una sola sede en Colombia (menos de 20 empleados).

Propósito: Centralizar la gestión de inventario farmacéutico por
lotes (método FEFO), ventas presenciales y en línea, servicio de
domicilios con cálculo automático de tarifas, gestión de clientes
y reportes gerenciales. El sistema debe cumplir con la normativa
colombiana vigente (Resolución 1403/2007, Decreto 677/1995 INVIMA,
Ley 2300/2023, Ley 1581/2012 Habeas Data).

Módulos del sistema:
1. Autenticación y control de acceso por roles (RBAC)
2. Gestión de clientes — registro con consentimiento Ley 1581,
   múltiples direcciones de entrega, historial de pedidos
3. Gestión de productos farmacéuticos — catálogo con principio
   activo, laboratorio/proveedor, código INVIMA (obligatorio),
   tipo de venta (libre / control especial), stock mínimo,
   precio compra/venta, forma farmacéutica, unidad de medida
4. Control de inventario por lotes — registro de entradas con
   número de lote, fecha de vencimiento, proveedor; método FEFO;
   alertas automáticas de stock mínimo y vencimiento < 30 días
5. Ventas presenciales (POS) — carrito, validación de fórmula
   médica para control especial, comprobante digital, FEFO
6. E-commerce (tienda en línea para clientes) — catálogo
   filtrado (sin control especial), carrito, checkout,
   cálculo de domicilio, pago por MercadoPago (PSE/Nequi/tarjeta)
7. Domicilios — seguimiento de pedidos, estados, panel repartidor
8. Reportes y dashboard gerencial — KPIs, exportación PDF/Excel

Roles de usuario:
- Administrador: acceso total, gestión de usuarios, configuración
  del sistema (tarifas domicilio, zonas de cobertura, stock mínimo)
- Gerente del local: dashboard operativo, supervisión de personal,
  gestión de devoluciones, reportes
- Auxiliar / Gestor de stock: CRUD de productos, registro de lotes,
  gestión de alertas y proveedores
- Vendedor: ventas presenciales POS, consulta de catálogo
- Repartidor: lista de pedidos asignados, actualización de estados
- Cliente: catálogo en línea, carrito, checkout, historial de pedidos

- Base de datos: MySQL 8.0 (13 tablas, 3FN)

Layout general (panel interno — roles empleados):
- Desktop (≥1024px): Sidebar fijo izquierdo 240px + Topbar 64px
  + área de contenido flex
- Tablet (768–1023px): Sidebar colapsable con toggle
- Mobile (<768px): Bottom navigation bar + sidebar como drawer

Layout tienda (vista cliente externo):
- Navbar superior con carrito flotante
- Grid de productos responsivo
- Flujo de checkout en pasos

LEYES UX APLICADAS — RESPETAR EN EL DISEÑO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Ley de Hick: máximo 5 ítems por sección del sidebar, formularios
  sin campos innecesarios visibles, checkout en pasos progresivos
- Ley de Fitts: botones de acción principal grandes (mín 48px alto)
  en posición predecible (esquina superior derecha o pie de formulario)
- Ley de Jakob: sidebar izquierdo, tabla con acciones a la derecha,
  formularios con flujo top-down, carrito de compras a la derecha
- Gestalt (proximidad): campos agrupados por secciones con separadores
  visuales; productos agrupados por categoría en la tienda
- Ley de Miller: máximo 10 filas por tabla con paginación; máximo
  12 productos por página en catálogo
- Visibilidad del estado: badges de estado siempre visibles, toasts
  para feedback de acciones, barra de progreso en checkout,
  indicador de stock en tiempo real en POS

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- HTML5 semántico (nav, main, section, article, header, aside)
- CSS con variables (custom properties) definidas en :root
- Importar Inter y JetBrains Mono desde Google Fonts
- Importar Lucide Icons desde CDN: https://unpkg.com/lucide@latest
- Responsivo con media queries en 768px y 1024px
- Sin frameworks externos — solo HTML, CSS y JS vanilla mínimo
- Cada pantalla en un archivo HTML independiente
- Comentarios CSS explicando cada sección principal
- El sidebar debe tener el ítem correspondiente resaltado
  como activo en cada pantalla, siguiendo esta estructura:
    Principal: Dashboard
    Comercial: Clientes / Pedidos / Ventas POS / Domicilios
    Inventario: Productos / Lotes / Alertas / Proveedores
    Análisis: Reportes
    Sistema: Usuarios / Configuración
- Los números de comprobante, INVIMA, lotes y referencias
  deben usar font-family JetBrains Mono
- Datos de ejemplo deben ser colombianos reales:
  nombres, ciudades de Huila (Neiva, Pitalito, Garzón),
  productos farmacéuticos reales (Amoxicilina Mk, Loratadina,
  Metformina Genfar, Omeprazol, Cetirizina Bayer, etc.),
  precios en pesos colombianos (COP)
